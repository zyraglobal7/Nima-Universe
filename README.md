# Nima Universe

Monorepo for the Nima product. One Convex backend, two clients.

## Layout

```
apps/
  native/       Expo / React Native app (nima-native)
  web/          Next.js app (nima-web)
packages/
  backend/      Convex backend (nima-backend) — single source of truth
```

All three workspaces share one Convex deployment (`gallant-tortoise-992` dev). The apps import generated types via `@/convex/_generated/...`, which is remapped in each app's `tsconfig.json` to `packages/backend/convex/*`.

## First-time setup

```bash
# 1. Install all workspace deps
npm install

# 2. Create env files from the examples
cp packages/backend/.env.local.example packages/backend/.env.local   # already exists; verify
cp apps/web/.env.local.example apps/web/.env.local                    # fill in secrets
cp apps/native/.env.local.example apps/native/.env.local              # fill in secrets

# 3. Generate Convex types (required before apps will typecheck)
npm run dev:backend
# ^ Ctrl-C once you see "Convex functions ready!"
```

## Daily dev

```bash
npm run dev:backend    # Convex dev server — leave running
npm run dev:web        # Next.js on :3000
npm run dev:native     # Expo dev server
```

Run `dev:backend` in its own terminal; apps read types from `packages/backend/convex/_generated/` which is regenerated by `convex dev`.

## Deploying the backend

```bash
npm run deploy:backend
```

## Testing

`apps/native` has a Jest + React Native Testing Library suite (`npm run test --workspace=apps/native`). A Husky `pre-push` hook runs it automatically and blocks `git push` if any test fails — it's wired up by the root `prepare` script, so a plain `npm install` after cloning is enough to activate it.

## Notes

- `packages/backend/convex/_generated/` is gitignored. Fresh clones must run `npm run dev:backend` once before `dev:web` or `dev:native` will typecheck.
- The native app's `metro.config.js` is configured for monorepo workspaces (`watchFolders`, `nodeModulesPaths`, `disableHierarchicalLookup`). Don't remove those lines.
- Sensitive files under `apps/native/` (signing credentials, Firebase admin JSON, `google-services.json`) are gitignored at the root. Provide them locally for Android/iOS builds.
